UTF-8 aware alternative to str_ireplace.

Case-insensitive version of str_replace This function is not fast and gets slower if $search/$replace is array. It assumes that the lower and uppercase versions of a UTF-8 character will have the same length in bytes which is currently true given the hash table to strtolower.
